   This does a fast LOCAL keyword check (no Claude API call) on the feature title and description. If it matches any of these patterns, return an instant classification WITHOUT calling Claude:

   TIER 1 — AUTO-SKIP (importance 1, no Claude call):
   Keywords in title or description (case-insensitive):
   - "fix", "bugfix", "bug fix", "hotfix", "patch"
   - "BE only", "backend only", "server only"
   - "investigate", "investigation", "debug", "debugging"
   - "deprecate", "deprecated", "remove", "removal"
   - "login", "auth", "authentication", "SSO", "OAuth" (login infrastructure)
   - "API key", "API token", "rate limit"
   - "refactor", "cleanup", "clean up", "housekeeping"
   - "CI/CD", "pipeline", "deploy script", "build fix"
   - "dependency", "dependencies", "upgrade deps", "bump version"
   - "typo", "spelling", "copy fix"
   - "revert", "rollback"
   - "test", "tests", "test fix", "flaky test"
   - "monitoring", "alerting", "logging"
   - "migration", "migrate" (DB migrations)

   Return format for auto-skipped:
   {
       "category": "auto_detected_category",
       "categories": ["bug_fix"] or ["infrastructure"] etc,
       "importance_score": 1,
       "is_user_facing": false,
       "target_audience": [],
       "marketing_summary": "Auto-classified: [matched keyword]. Use override to reclassify if needed.",
       "recommended_channels": ["notion_monthly"],

   - BE only/backend/API/refactor/CI/dependency/migration/monitoring → "infrastructure"

   - investigate/debug → "infrastructure"
   - login/auth/SSO → "infrastructure"
   - test/flaky → "infrastructure"

   TIER 2 — FULL CLAUDE CLASSIFICATION (everything else):
   If quick_classify returns None (no keyword match), proceed with the full Claude API call as before.

2. UPDATE the classification pipeline:

   - If it returns None, proceed with Claude classification as normal
