1. CREATE `integrations/sendgrid_client.py`:
   - Install sendgrid package (add to requirements.txt)
   - Function `send_email(subject: str, body: str, to_email: str = None, is_test: bool = True) -> dict`:
     - Check for SENDGRID_API_KEY and SENDGRID_FROM_EMAIL in environment variables
     - If credentials exist:
       - If is_test: send to SENDGRID_TEST_EMAIL env var (or to_email if provided), with "[TEST] " prefix on subject
       - Build email using sendgrid Mail helper
       - Send via SendGrid API
       - Return {"success": true, "message_id": "...", "method": "sendgrid", "to": to_email, "is_test": is_test}
     - If credentials don't exist:
       - Return fallback: {"success": true, "method": "fallback", "message": "Email draft ready — SendGrid not configured", "subject": subject, "body": body, "preview_html": rendered_preview}
     - On error: return {"success": false, "error": str(e)}
   - Function `render_email_html(subject: str, body: str) -> str`:
     - Takes the plain text body and wraps it in a clean, minimal HTML email template
     - Chartmetric branded: dark header bar with "Chartmetric" text, white content area, teal CTA button if body contains "Try it here" or similar CTA phrase
     - Keep the HTML email template simple and inline-styled (email clients don't support external CSS)
